# Handoff: Context Explorer — Real Session Mode & Polish

**작성일**: 2026.04.08
**브랜치**: `feature/context-explorer`
**상태**: 완료 · 릴리스 준비

---

## TL;DR

Context Window Explorer에 **실제 세션 재생 모드**를 추가했고, 이후 여러 라운드의 UX 다듬기 — 콤보박스 검색, 기간 필터 연동, 동적 컨텍스트 예산(200K/1M), 범례 퍼센트 툴팁, 새로고침 상태 유지 등 — 를 적용했다. 원본 시뮬레이터 기반이었던 페이지가 이제 사용자의 실제 하네스 사용 내역을 바로 확인할 수 있는 페이지로 확장됐다.

---

## 주요 변경 사항

### 1. 실제 세션 모드 (`buildSessionEvents`)

- `tokenEntries` → 타임라인 이벤트 변환. delta, cumulative, 캐시 비율, 모델, 타임스탬프 포함
- `promptStats`와 ±2초 매칭하여 `kind='user'` 분류
- 급격한 음수 delta 감지 → `kind='compact'` (/compact 감지)
- 예시 모드와 공존하는 모드 토글 탭 (`예시 세션` / `실제 세션`)

### 2. 세션 검색 콤보박스

- `<select>` → `<input>` 기반 autocomplete 콤보박스
- 포커스/클릭 시 팝오버 목록, 입력 시 실시간 필터
- 항목 포맷: 첫 사용자 프롬프트 50자 + 날짜/시간 + turn 수
- Focus 시 기존 값 stash+비우고, blur 시 새 선택 없으면 복원
- 세션 선택은 module-level `contextSubPath`에 저장 → 재렌더 시 자동 복원
- `listReplayableSessions()` + `filteredSessions()` (검색·정렬·기간 3종 결합)

### 3. 정렬 & 필터

- 정렬 버튼: **최근순** / **턴 많은순**
- 기간 필터: 좌측 사이드바의 전역 `currentPeriod`/`customDateRange`와 연동
- "턴 많은순" 버튼에 CSS 툴팁으로 "턴" 의미 설명 (`cwe_turnsHelp`)

### 4. 동적 컨텍스트 예산 (200K / 1M)

- `state.budget = peak > 200K ? 1_000_000 : 200_000`
- 헤더 sub label 포맷: `/ 1M (38%) · opus-4-6 · measured`
  - budget
  - 프로그레스 퍼센트
  - 세션에서 사용된 unique 모델명 (`<synthetic>` 필터링)
  - measured/illustrative 태그
- `?` 도움말 아이콘: 버젯이 세션별로 달라지는 이유 설명

### 5. 헤더 토큰 = peak cumulative + 스택 바 정규화

- `totalTokens`를 "last visible cumulative"에서 "**max cumulative**"로 변경 (중간 /compact 이후에도 peak 유지)
- `renderBarSegments`는 session 모드에서 `sessionScale = peak / sumDeltas` 적용 → 스택 바 총 폭 = 헤더 숫자와 정확히 일치
- session 모드에서 min-width 0.15% 클램프 제거 (600+ 블록의 거품 인플레이션 방지)

### 6. 범례 퍼센트 → floating 툴팁

- 범례 호버 시 `document.body`에 붙은 floating 툴팁으로 `"시스템: 12% · 73K tokens"` 표시
- z-index 10000 + viewport 클램프로 `.cw-root { overflow: hidden }` 클리핑 회피
- 좌/우 가장자리 legend item에서도 잘리지 않음

### 7. 세션 모드 Kind 뱃지 분리

원본: skill/mcp/tool → 전부 `kind='claude'` 로 매핑되어 모든 항목이 "claude" 뱃지
수정: 각 컨텍스트에 고유 `kind` 할당

- `skill` (노랑 #B8890B)
- `mcp` (보라 #9B7BC4)
- `agent` (보라 #9B7BC4)
- `tool` (무채 #6E6C64)

신규 i18n 키: `cwe_kind{Skill,Mcp,Agent,Tool}` + `*Detail`

### 8. Promptstats 프리뷰

- `scripts/parsers/usage.mjs` 파서에 `preview` 필드 추가 (첫 60자)
- tool-result / system-reminder pseudo-prompt 필터링

### 9. URL 해시 기반 상태 지속

- `contextSubPath` (module-level) = `'' | 'session' | <sessionId>`
- `getHash()` 확장: `'#context/' + contextSubPath` 반환
- `applyHash()` / `setMode()` / `selectSession()` 모두 `contextSubPath` 갱신
- 새로고침 / 스코프 변경 / 기간 변경 시 세션 상태 유지

### 10. 레이아웃 & 스타일

- `.cw-root` 높이: `calc(100vh - 64px)` + `min-height: 850px` (상하 여백 32px 균형)
- 상단 top + 하단 bottom 패딩 대칭 (16px)
- 세션 모드에서 하단 재생 컨트롤 바 숨김 (`display: none`)
- 세션 모드에서 빈 상태 hint ("세션을 선택하세요")
- fmt() 1M 단위 지원
- terminal visibility eye 아이콘 `stroke-width`: full=3.5, brief=3

### 11. 예시 모드 정리

- 게이트 카드의 `<input>` 편집 기능 제거 → 고정 텍스트 표시 (예시 시나리오에서는 수정이 의미 없음)
- 관련 state/handlers (`typedTexts`, `gateFocusWanted`, `replayMode`, `cwEstimateTokens`, input 이벤트) 모두 제거

### 12. 툴팁 시스템 (`.cw-tip`)

- CSS `::after` + `attr(data-tip)` 기반 툴팁
- `.cw-tip-right` 변종 — 우측 정렬 (컨테이너 오른쪽 끝 요소용)
- `white-space: pre-wrap` — 여러 줄 지원
- 0.25s 호버 딜레이, 다크 모드 대응
