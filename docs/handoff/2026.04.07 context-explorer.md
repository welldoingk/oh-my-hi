# Handoff: Context Window Explorer

**작성일**: 2026.04.07
**브랜치**: `feature/context-explorer`
**상태**: 핵심 기능 완성 · 추가 다듬기 필요

---

## TL;DR

oh-my-hi 대시보드에 **Context Window Explorer** 페이지를 새로 추가했다. Claude Code docs의 [Explore the context window](https://code.claude.com/docs/en/context-window) 인터랙티브 시뮬레이터를 포팅했지만, 고정 시나리오가 아니라 **현재 하네스 설정에서 실측한 startup 컨텍스트 값**을 보여주고, **사용자가 직접 프롬프트를 입력**해서 세션 흐름을 체험할 수 있도록 확장했다.

## 완료된 작업

### 1. 1-depth 사이드바 메뉴 `#context`

- `templates/app.js`
  - `applyHash()` — `context` 뷰 라우팅 등록
  - `renderSidebar()` — 🪟 "컨텍스트 익스플로러" nav 항목 추가 (Structure 아래)
  - `renderContent()` — `context` 분기에서 `renderContextExplorer()` 호출
- `templates/locales/{en,ko}.json` — `contextExplorer` 키 추가

### 2. 원본 충실 포팅 + vanilla JS 리팩터링

원본 React 컴포넌트를 vanilla JS 단일 함수 `renderContextExplorer()`로 포팅. `templates/app.js` 3317~3990 라인 근처. 주요 구성:

- **EVENTS** — 36개 이벤트 (id 기반, i18n 키 참조)
- **GATES** — 5개 게이트 (id 기반)
- **KIND_META / VIS_META / LEGEND** — i18n 키 기반
- **state** — 클로저 내 mutable 객체
- **update()** — 전체 sub-region 재렌더
- **startAnim()** — `requestAnimationFrame` 애니메이션 루프 (`0.032` time/sec)
- **cleanup** — `window._cwRafId`, `_cwKeyHandler`, `_cwFsHandler`로 뷰 전환 시 정리

### 3. 실측 데이터 주입 (`scripts/generate-dashboard.mjs`)

- `estimateTokens(text)` — ASCII 0.25/char, 비ASCII 0.55/char 가중치
- `computeContextStats(scopeData, globalConfigDir, scope)` — 스코프별 계산:
  - `globalClaudeTokens` — `~/.claude/CLAUDE.md`
  - `projectClaudeTokens` — 프로젝트 루트의 CLAUDE.md (프로젝트 스코프만)
  - `autoMemoryTokens` — `<configPath>/memory/MEMORY.md`
  - `skillsCount` / `skillsDescTokens` — 모든 skill description 합 + 스킬당 ~10 토큰 오버헤드
  - `mcpServersCount` / `mcpToolsTokens` — MCP 서버 수 × 15
- `collectScopeData` / `collectProjectData`가 각 scope data에 `contextStats` 주입
- `collectAllScopes` 후처리 단계에서 프로젝트 스코프가 글로벌 스킬/MCP를 **backfill** — 프로젝트 작업 시 실제 로드되는 startup context를 정확히 반영
- `emptyScopeData()`에도 `contextStats: null` 포함

### 4. i18n 전체 이주 (165개 신규 키)

`templates/locales/{en,ko}.json`에 prefix `cwe_`로 추가.

- UI 라벨 (~33): `cwe_title`, `cwe_subtitle`, `cwe_startSession`, `cwe_keyTakeaway`, `cwe_terminalView`, `cwe_afterCompactTitle`, `cwe_afterCompactStats` (args 3), `cwe_subagentReturned` (args 1), `cwe_tokensInSubagent` (args 1), ...
- Kind meta (12): `cwe_kindAuto` ~ `cwe_kindSubDetail`
- Vis meta (6): `cwe_visHidden` ~ `cwe_visFullSub`
- Legend (11): `cwe_legSystem` ~ `cwe_legHooks`
- Events (~77): `cwe_ev{1..36}_label/_desc`, `cwe_ev{7,9,18,23,35}_tip`, `cwe_ev0_label/_desc` (synthetic summary block)
- Gates (5): `cwe_gate1` ~ `cwe_gate5`
- Takeaways (10): `cwe_take_compacted` ~ `cwe_take_compact`
- Terminal views (9): `cwe_term_compacted` ~ `cwe_term_full`

모든 키는 en/ko 매칭, 빈 값 없음 → `test/web-ui.test.mjs`의 locales 테스트 통과.

### 5. 이벤트 데이터 구조 리팩터링

```js
// 원본 (고정 라벨/설명/tokens)
{ t: 0.12, kind: 'auto', label: '~/.claude/CLAUDE.md', tokens: 320, desc: '...', ... }

// 개선 (id + statKey)
{ id: 6, t: 0.12, kind: 'auto', tokens: 320, color: '#6A9BCC', vis: 'hidden',
  statKey: 'globalClaudeTokens', link: '...' }
```

헬퍼:
- `evtLabel(e)`, `evtDesc(e)`, `evtTip(e)` — `typedFor(e.id) || t('cwe_ev' + e.id + '_label')`
- `resolveStat(statKey)` — `mcpPlusSkills` 특수 키는 `mcpToolsTokens + skillsDescTokens` 반환
- `evtTokens(e)` — 실측값 오버라이드 (sub events는 `evtSubTokens`)
- `cwEstimateTokens(text)` — 클라이언트 사이드 토큰 추정 (빌드 스크립트와 동일 공식)

실측값 오버라이드 이벤트:
- id 2 (Auto memory) → `autoMemoryTokens`
- id 4 (MCP tools deferred) → `mcpToolsTokens`
- id 5 (Skill descriptions) → `skillsDescTokens`
- id 6 (~/.claude/CLAUDE.md) → `globalClaudeTokens`
- id 7 (Project CLAUDE.md) → `projectClaudeTokens`
- id 26 (sub: Project CLAUDE.md own copy) → `projectClaudeTokens` (subStat: true)
- id 27 (sub: MCP + skills) → `mcpPlusSkills` (subStat: true)

나머지 이벤트(시스템 프롬프트·환경 정보·파일 읽기·훅·Claude 작업 등)는 illustrative 값 유지.

### 6. 편집 가능한 프롬프트 게이트

게이트 카드의 고정 텍스트를 `<input type="text">`로 교체해서 사용자가 직접 프롬프트를 입력할 수 있게 함.

- `GATE_EVENT_ID = [8, 23, 34, 35, 36]` — 게이트 인덱스 ↔ 이벤트 id 매핑
- `state.typedTexts` — 게이트 인덱스별 입력값 저장
- `state.gateFocusWanted` — 게이트 첫 등장 시 한 번만 자동 포커스
- `update()`에서 active input의 `value`/`selectionStart`/`selectionEnd` 저장 → renderTimeline 후 복원 (타이핑 중 hover/click으로 update가 호출돼도 커서가 유지됨)
- Enter 키로 Send, 클릭 시에도 `sendGate()` 호출
- 이벤트 8·23(Your prompt / Your follow-up)은 입력값으로 토큰 재추정 (`cwEstimateTokens`)
- 34~36(!git status, /commit-push, /compact)은 명령+출력 합산이라 기본값 유지

## 남은 작업 / 개선 아이디어

### 우선순위 ★★★

- [ ] **시나리오 이벤트 한국어 표현 다듬기** — 이벤트 8~36의 파일 읽기 / Claude 작업 / 훅 관련 desc를 실제로 검토하며 문맥에 맞게 다듬을 것. 현재는 기계적 번역 수준
- [ ] **실측 가능한 토큰과 시나리오 토큰의 구분을 UI에서 명확히** — 현재 `illustrative` 뱃지나 라벨이 충분히 눈에 띄지 않음. takeaway 패널 상단에 "이 세션은 실측 + 시나리오가 섞여 있습니다" 같은 알림 배너 추가 검토

### 우선순위 ★★

- [ ] **`work/` 프로젝트 스코프에서 skills/MCP가 0인 이슈 재확인** — `-Users-netil-dev-oh-my-hi` 스코프는 글로벌 backfill이 적용되지만, `work/` 같은 다른 프로젝트도 동일하게 적용되는지 full rebuild 후 검증
- [ ] **토큰 추정 정확도 개선** — 현재 `cwEstimateTokens`는 대략값. `tiktoken` 포팅은 과하지만, 코드 비율이 높은 텍스트에 대해 소폭 보정 추가 고려
- [ ] **MEMORY.md가 없는 프로젝트의 시각화** — 현재 `autoMemoryTokens: 0`이면 이벤트 자체가 바 차트에서 사라짐 (막대 폭 0.15% 최소값은 있음). 0일 때 "없음" 표시를 별도로 할지 결정
- [ ] **"새 프롬프트로 다시 시작" 버튼** — 현재는 완주 후 ↺(restart)를 누르면 state가 리셋되지만 `typedTexts`도 리셋됨. 프롬프트만 유지하고 replay 할 수 있게 할지 검토

### 우선순위 ★

- [ ] **키보드 단축키 힌트** — Space(재생/일시정지), Enter(게이트 전송) 힌트를 어딘가 노출
- [ ] **input placeholder 활용** — 현재 value에 기본 프롬프트를 프리필하는데, 사용자가 전부 지운 뒤 의도를 표시하기 위해 placeholder도 설정해두는 것이 좋음
- [ ] **CHANGELOG.md / SPEC.md / GUIDE.md 업데이트** — 아직 작업 안 함. 릴리스 전에 반영 필요
- [ ] **버전 bump** — 릴리스 시 `0.4.8` 또는 `0.5.0` (feature scope에 따라). 사용자가 릴리스 명시 요청 시까지 대기
- [ ] **스크린샷/GIF** — README에 추가
